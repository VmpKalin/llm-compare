To solve this problem, we need to merge overlapping or touching intervals from a given list of intervals. The goal is to return a list of non-overlapping intervals sorted by their start times.

### Approach
1. **Sort Intervals**: First, we sort the intervals based on their start times. This helps in easily identifying and merging overlapping intervals as they will be adjacent after sorting.
2. **Initialize Merged List**: We initialize a list with the first interval from the sorted list.
3. **Iterate Through Intervals**: For each subsequent interval, we compare it with the last interval in our merged list:
   - If the current interval starts before or at the end of the last merged interval, they overlap or touch, so we merge them by updating the end time to be the maximum of both intervals' end times.
   - If the current interval does not overlap with the last merged interval, we add it as a new entry in our merged list.

This approach ensures that all overlapping and touching intervals are efficiently merged into non-overlapping intervals.

### Solution Code
```python
def merge(intervals):
    if not intervals:
        return []
    sorted_intervals = sorted(intervals, key=lambda x: x[0])
    merged = [sorted_intervals[0]]
    for current in sorted_intervals[1:]:
        last = merged[-1]
        if current[0] <= last[1]:
            last[1] = max(last[1], current[1])
        else:
            merged.append(current)
    return merged
```

### Explanation
- **Sorting**: The intervals are sorted by their start times to ensure that any overlapping or touching intervals are adjacent.
- **Merging Process**: We iterate through each interval, starting from the second one. For each interval, we check if it overlaps with the last interval in our result list. If it does, we merge them; otherwise, we add the current interval as a new entry.
- **Efficiency**: This approach efficiently handles merging by only making a single pass through the sorted intervals after sorting, resulting in an overall time complexity of O(n log n) due to the sorting step.

This solution ensures that all overlapping or touching intervals are merged into non-overlapping intervals and returned in the correct order.
