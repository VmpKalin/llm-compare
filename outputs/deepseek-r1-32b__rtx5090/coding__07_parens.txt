To solve this problem, we need to generate all possible combinations of n pairs of well-formed parentheses. A well-formed parenthesis string is one where every opening parenthesis '(' has a corresponding closing parenthesis ')', and at no point does the number of closing parentheses exceed the number of opening ones.

### Approach
The approach to solve this problem involves using recursion with backtracking. The idea is to build each valid combination step by step, ensuring that we never have more closing parentheses than opening ones at any point in the string. Here's a detailed breakdown:

1. **Initialization**: Start with an empty result list to store all valid combinations.
2. **Recursive Helper Function**: Define a helper function that takes three parameters: the current string being built, the count of opening parentheses used so far, and the count of closing parentheses used so far.
3. **Base Case**: If both the counts of opening and closing parentheses reach n, add the current string to the result list as it is a valid combination.
4. **Recursive Steps**:
   - If the number of opening parentheses is less than n, recursively call the helper function with an additional opening parenthesis.
   - If the number of closing parentheses is less than the number of opening ones, recursively call the helper function with an additional closing parenthesis.

This approach ensures that we explore all possible valid combinations by building them step by step and backtracking when necessary.

### Solution Code
```python
def generate(n):
    result = []
    def backtrack(s, open_p, close_p):
        if len(s) == 2 * n:
            result.append(s)
            return
        if open_p < n:
            backtrack(s + '(', open_p + 1, close_p)
        if close_p < open_p:
            backtrack(s + ')', open_p, close_p + 1)
    backtrack('', 0, 0)
    return result
```

### Explanation
- **Initialization**: The function `generate` initializes an empty list `result` to store the valid combinations.
- **Helper Function**: The nested function `backtrack` is defined to build each combination. It takes three parameters: the current string `s`, the count of opening parentheses `open_p`, and the count of closing parentheses `close_p`.
- **Base Case**: When the length of the current string reaches `2 * n`, it means we have used all n pairs, so we add this string to the result list.
- **Recursive Steps**:
  - If there are fewer than n opening parentheses, we add another '(' and recurse.
  - If there are fewer closing parentheses than opening ones, we add a ')' and recurse.

This method efficiently explores all valid combinations by ensuring that each step maintains the balance of parentheses, thus generating all possible well-formed strings.
