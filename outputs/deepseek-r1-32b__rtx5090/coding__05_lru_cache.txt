To solve this problem, we need to implement an LRUCache class in Python with a fixed capacity. The methods `get` and `put` must both run in O(1) average time complexity. We will use the `OrderedDict` from the `collections` module to efficiently manage the least recently used items.

### Approach
The approach leverages the properties of `OrderedDict`, which maintains insertion order by default. When an item is accessed or added, it can be moved to the end using the `move_to_end` method, marking it as the most recently used. The beginning of the ordered dictionary will always contain the least recently used items.

1. **Constructor**: Initializes the cache with a given capacity and sets up an empty `OrderedDict`.
2. **Get Method**: Checks if the key exists in the cache. If it does, the key is moved to the end (marking it as most recently used), and its value is returned. If not, returns -1.
3. **Put Method**: Updates or adds a key-value pair. If the key already exists, its value is updated, and the key is moved to the end. If the cache exceeds its capacity, the least recently used item (at the beginning) is removed before adding the new item.

### Solution Code
```python
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity):
        self.capacity = capacity
        self.cache = OrderedDict()

    def get(self, key):
        if key in self.cache:
            self.cache.move_to_end(key)
            return self.cache[key]
        else:
            return -1

    def put(self, key, value):
        if key in self.cache:
            self.cache[key] = value
            self.cache.move_to_end(key)
        else:
            if len(self.cache) >= self.capacity:
                self.cache.popitem(last=False)
            self.cache[key] = value
```

### Explanation
- **Initialization**: The constructor initializes the cache with a specified capacity and an empty `OrderedDict`.
- **Get Operation**: When retrieving a key, it checks for existence. If found, it moves the key to the end of the dictionary (marking it as recently used) and returns its value. If not found, it returns -1.
- **Put Operation**: When adding or updating a key-value pair, if the key exists, it updates the value and moves the key to the end. If the cache is full, the least recently used item (first in the dictionary) is removed before adding the new item.

This approach ensures that both `get` and `put` operations are efficient, operating in O(1) average time complexity due to the properties of `OrderedDict`.
