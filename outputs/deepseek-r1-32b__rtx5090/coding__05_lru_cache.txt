To solve this problem, we need to implement an LRUCache class in Python with a fixed capacity. The class should support two main operations: `get(key)` and `put(key, value)`, both of which must run in O(1) average time complexity. 

### Approach
The approach leverages the `collections.OrderedDict` data structure, which maintains insertion order and allows efficient access to the least recently used items. Here's a detailed breakdown:

1. **Initialization**: The constructor initializes an `OrderedDict` to store cache entries and sets the capacity.
2. **Get Operation**: 
   - If the key is not present in the cache, return -1.
   - If the key exists, update its position to mark it as recently used by moving it to the end of the ordered dictionary and return its value.
3. **Put Operation**:
   - If the key already exists, update its value and move it to the end to mark it as recently used.
   - If the key does not exist, add it to the cache. If adding this new entry exceeds the capacity, remove the least recently used item (the first item in the ordered dictionary).

### Solution Code
```python
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity):
        self.capacity = capacity
        self.cache = OrderedDict()

    def get(self, key):
        if key not in self.cache:
            return -1
        # Move the accessed key to the end (most recently used)
        self.cache.move_to_end(key)
        return self.cache[key]

    def put(self, key, value):
        if key in self.cache:
            # Update the value and move it to the end
            self.cache[key] = value
            self.cache.move_to_end(key)
        else:
            # Add new entry
            self.cache[key] = value
            # Check if we need to evict the least recently used item
            if len(self.cache) > self.capacity:
                # Remove the first key (oldest)
                oldest_key = next(iter(self.cache))
                del self.cache[oldest_key]
```

### Explanation
- **Initialization**: The constructor sets up an `OrderedDict` and stores the capacity.
- **Get Method**: Checks if the key exists. If it does, the method moves the key to the end of the dictionary (marking it as recently used) and returns its value. If not found, it returns -1.
- **Put Method**: Updates or adds a new entry. If adding a new entry exceeds the capacity, the least recently used item (the first item in the ordered dictionary) is removed.

This approach ensures that both `get` and `put` operations run efficiently in O(1) average time complexity, leveraging the properties of `OrderedDict` to maintain insertion order and manage cache eviction effectively.
